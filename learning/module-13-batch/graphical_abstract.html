Cancellation — the one subtle decision

class BatchCancelled(BaseException) — not Exception.

  • Services wrap each molecule in except Exception so one bad structure can't abort a run
  • A cancel raised inside such a loop must not be mistaken for a bad molecule
  • BaseException (the KeyboardInterrupt family) sails through those handlers; only run_batch catches it → stops, returns a partial cancelled=True result
  • Raised via a wrapped progress callback → no service changed, cancellation is coarse (per progress tick) but honest
