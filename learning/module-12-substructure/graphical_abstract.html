The renderer change (small, shared)

render_svg(mol, …, highlight_atoms=None) — one optional arg. It computes the bonds within the matched atom set so a ring reads as one region, and passes highlightAtoms+highlightBonds to RDKit.

  • Input mol still never mutated (draws a copy — tested)
  • Default path byte-for-byte unchanged
  • Lives in services → lights up both views, zero per-widget drawing code

Highlighting through the thumbnail cache

The fix for stale thumbnails: fold the match into the cache key.

key = (canonical SMILES, sorted matched-atoms tuple | None)

Same molecule caches separately with/without a highlight. New search → new key → re-render; clear query → key reverts → plain image from cache. Invalidation falls out of the key — no explicit clearing, no stale frames.

Key Ideas

Design
  • A match is relational — SubstructureHit carries its query; stale hits cleared (3rd time this pattern appears)
  • searched-no-match (—) ≠ unsearched (blank) — an honesty distinction the sort respects
  • 4th filter channel — refactored proxy to all(channel.matches …); cleared on standardize
Reuse (almost everything)
  • render_svg · delegate · Structure panel · filter proxy · worker
  • parse_query mirrors parse_smiles (mute stderr, return None)
  • live-validation dialog pattern from similarity
  • Only new surface: the highlight itself
