Similarity Search — Finding More Like This One

Pick a query molecule, rank the whole dataset against it, and never trust a number whose context you can't see. The payoff for Module 6's bit vectors.

The Core Idea

Tanimoto(A, B) = |A ∧ B| / |A ∨ B|  →  one query, one C++ call, a ranked list
BulkTanimotoSimilarity(query, [all]) crosses the Python↔C++ boundary once for the entire dataset and loops in C++ — not once per molecule. This is why Module 6 kept RDKit's native ExplicitBitVect instead of converting to numpy: bulk eats it directly, no conversion step.

The Design Fork: a similarity is not a property of a molecule

Intrinsic — Modules 5 & 6

MW = 180.16
  • Depends on the structure and nothing else.
  • Recompute tomorrow → same answer.
  • A bare number is a complete fact.

Relational — Module 7

0.448  ←  meaningless alone
  • 0.448 against aspirin, by Tanimoto, on Morgan r2 · 2048b.
  • Change any of those three and the number changes — while looking just as plausible.
  • So SimilarityScore refuses to be a bare float: it carries its query.
Same discipline as Module 6's Fingerprint carrying its options: a derived number travels with the context that makes it true. The table cell shows 0.448; the tooltip shows "Tanimoto vs aspirin · Morgan r2 · 2048b". You cannot read the score without its context being one hover away.

The Bug This Module Exists To Prevent

RDKit guards exactly one thing: bit length. Aspirin vs salicylic acid, all measured:

ComparedSame length?RDKit saysVerdict
Morgan r2 vs Morgan r2yes0.448the truth
Morgan 2048 vs MACCS 167noValueErrorloud, obvious, safe
Morgan r2 vs Morgan r3yes0.394silent · plausible · wrong
Morgan r2 vs RDKit pathyes0.005silent · a real hit looks like noise
ECFP vs FCFPyes0.000silent · a close analogue scores zero
The mistake RDKit catches is the one you'd have noticed anyway. The mistake that ships is same length, different meaning — every number above is a float in exactly the range a chemist expects, and only the first means anything. is_comparable_to returns False for all three silent rows.
  • Step 1 — encode the query with the same options as the dataset, never with defaults.
  • Step 2 — check every record with is_comparable_to anyway, and refuse the failures.
  • Step 2 looks redundant after step 1. It isn't — and we only found out by writing the test.
  • compute_fingerprints overwrites a vector only on success. A molecule RDKit throws on keeps its old fingerprint.
  • Re-encode a MACCS dataset as Morgan with one failure → exactly one record is still MACCS. Same field, same type, different meaning.
Cost: one equality check per record. Prevents: a plausible, wrong hit list — the kind of bug that gets a compound synthesized for no reason. Skipped rows go blank, never 0.000: blank means "not compared", 0.000 claims "compared, nothing in common".

Three Metrics, One Question

Tanimoto default

c / (a + b − c)
Overlap ÷ union. The field standard — every published "≥ 0.85" threshold assumes it. If unsure, this.

Dice

2c / (a + b)
Counts overlap twice → always ≥ Tanimoto (we test this). Fairer when the two molecules differ a lot in size.

Cosine

c / √(a·b)
The angle between the vectors. Common in chemical-space work and ML.

Two Decisions Worth Stealing

No threshold field on the dialog

  • Most tools ask "cutoff?" before you've seen a single score. That's a guess: too high and you re-run, too low and you scroll.
  • Scores land in a sortable column; the Module 5 filter box narrows them afterwards — Sim >= 0.7, interactively, at whatever cutoff turns out to be interesting.
  • One box for MW < 500 and Sim >= 0.7. One grammar. Reuse worth having removes a control, not just code.

Extract on the second consumer, not the first

  • Module 6 kept the fingerprint controls inside its dialog — correct: a widget invented for one caller is an abstraction you pay for and don't use.
  • Module 7 is the second caller, so FingerprintOptionsWidget was extracted now: one enable/disable rule, and the QVariant coercion can't be forgotten in the next dialog.
  • Extract when the duplication is real and present, not when it's imagined.

Found By Looking At The Screen

A checked radio button rendered nothing

Module 7 is the first to use radios. Qt's native style drew the checked dot invisibly on our backgrounds — unchecked showed a circle, checked showed nothing, so the dialog looked like no option was selected. Invisible to every test; obvious in one screenshot. Fixed with an explicit qradialgradient indicator in both themes.

The result landed off-screen

The Similarity column sits past the whole descriptor panel. On a normal window, running a search produced no visible change — sorting silently rearranged rows, which is somehow worse than nothing happening. refresh_similarity now ends by scrolling the score into view.
