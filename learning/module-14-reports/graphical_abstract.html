Report Generation — a Shareable Snapshot, HTML & PDF

One summary, two writers: self-contained HTML (inline SVG) and paginated PDF (ReportLab + Cairo PNG). Both off Qt.

One summary → two writers (identical numbers)

build_summary(records)stats over what's computed · honest about gaps
→
write_htmlinline <svg> · print CSS · zero deps
write_pdfReportLab · Cairo PNG · A4 paginated

The two files can never disagree — they read the same ReportSummary.

The renderer, extended once more

  • HTML embeds inline SVG — render_svg already produced it (crisp, no files)
  • PDF needs raster → new render_png via RDKit Cairo (MolDraw2DCairo) — no Qt, so the whole report runs in the worker
  • Both share a private _draw helper (copy mol · 2D coords · palette · optional highlight); differ only in drawer class + background
  • Refactored into _draw on the second renderer, not speculatively

Report contents

Summary
  • Title · timestamp · Wawekit version
  • Headline facts: molecules · Lipinski pass · clusters · scaffolds
  • Descriptor Min/Mean/Max table (only if computed)
Molecule grid
  • Depiction + name + formula + key properties per card
  • Capped for readability — stats still cover all
  • Base output path → name.html + name.pdf in one click
