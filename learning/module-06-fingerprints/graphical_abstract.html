Fingerprints — Turning Structures Into Comparable Bits

Encode each molecule as a bit vector so similarity becomes a bitwise operation. The engine room for similarity search (Module 7) and clustering (Module 11).

The Core Idea

aspirin → 0100…1001 (2048 bits, 24 set)
Tanimoto(A, B) = |A ∧ B| / |A ∨ B|   →   aspirin vs paracetamol = 0.222
A hard chemistry question becomes a fast bitwise one. DataStructs.BulkTanimotoSimilarity is exactly what Module 7 will call — which is why we keep RDKit's native ExplicitBitVect instead of converting.

The Real Design Work: Comparability

Morgan r=2 and r=3 are different encodings. Tanimoto between them returns a number that looks fine and means nothing — and nothing stops it happening by accident: compute, load more molecules, recompute with a different radius, and the dataset is quietly mixed. Silent wrongness is the worst failure mode in this project.

The fix
  • Every Fingerprint carries the options that built it
  • Cache rule: reuse only if params match (generalizes Module 5's "reuse if present")
  • Params changed ⇒ recompute, never mix
  • is_comparable_to() lets Module 7 assert before trusting a number
Normalization
  • MACCS ignores radius/bits/features → MACCS(r=2) and MACCS(r=5) are bit-identical and must compare equal
  • Otherwise Module 7 refuses a perfectly comparable dataset and the cache recomputes for nothing
  • Identity = the params that actually shaped the bits
Proof (fingerprints-maccs.png): after Morgan, recomputing as MACCS reports "9 computed, 0 reused" — every row recomputed because params changed.

Two Gotchas Worth Remembering

RDKit: the tutorials are outdated
  • AllChem.GetMorganFingerprintAsBitVect now prints "DEPRECATION WARNING: please use MorganGenerator"
  • Current API: rdFingerprintGenerator.GetMorganGenerator(...) — verified against the installed RDKit before writing code
  • Generator built once per run, returned as a callable — Module 4's perf lesson restated
Qt: StrEnum → QVariant → plain str
  • Combo item data round-trips a StrEnum back as a plain str
  • "MACCS" is FingerprintKind.MACCS → False; == → True
  • Symptom: dialog reported Morgan whatever you picked. Caught by driving it headlessly, not by reading code
  • Fix: coerce at the Qt boundary and use == in the model (also fixes Module 15's TOML strings). match/case was already safe — value patterns use ==
