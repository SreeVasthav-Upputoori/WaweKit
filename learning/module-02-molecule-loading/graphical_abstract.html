SDF / MOL / SMILES input with per-record error recovery, background threading, and Qt Model/View display.

Key Ideas

RDKit concepts
  • Mol object — wrapped in MoleculeRecord with name + provenance
  • SDMolSupplier — indexed SDF reader; None = failed record
  • Sanitization — valence/aromaticity validation; why records fail
  • GetPropsAsDict() — SDF data fields preserved on each record
  • Canonical SMILES — computed once, cached (views repaint often)
Qt concepts
  • QThreadPool + QRunnable — pooled background work, no manual threads
  • Cross-thread signals — auto-queued onto the GUI event loop; no locks
  • Model/View — data (model) separated from painting (view)
  • Sort proxy — sorts by raw UserRole values, so 100 > 20
  • Drag & drop, QProgressBar in the status bar

Engineering Decisions

LoadReport: errors are data, not exceptions loader is UI-free → unit-testable generic FunctionWorker (reused by 10+ future modules) services may use QtCore, never QtWidgets sequential load queue progress throttled to every 100 records verified: 18 tests · ruff · black
