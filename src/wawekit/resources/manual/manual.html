1. Welcome — what is WaweKit?

WaweKit is a desktop application for working with collections of molecules. You load a file of structures, and WaweKit helps you clean them, measure them, compare them, group them, view them in 3D, and report on them — the everyday toolkit of early drug discovery and cheminformatics research.

Everything runs on your computer: no account, no upload, no internet needed. Long operations run in the background, so the app never freezes — watch the progress bar in the bottom-right corner.

2. Quick start: your first 5 minutes

  1. Open a file. Use File → Open Molecules… (Ctrl+O) or simply drag a file onto the window. Try one of the demo files in the samples/ folder, e.g. demo_similarity.smi.
  2. Click any row. The Structure panel (right side) shows a large drawing of that molecule with its properties.
  3. Compute descriptors. Press Ctrl+D. New columns appear: MW, LogP, TPSA and more.
  4. Filter. Type MW < 300 in the filter box above the table and press Enter. Only light molecules remain. Clear the box to see everything again.
  5. Explore. Every other feature lives in the Chemistry menu — each one is described below.

3. The main window, piece by piece

  • Molecule table (centre) — one row per molecule: a structure thumbnail, name, SMILES, and a column for every value you compute later. Click a column header to sort. Right-click a row for copy/remove and quick similarity search.
  • Workspace panel (left) — a running log of what you've done: files loaded, standardizations, searches, batches, reports.
  • Scaffolds tab (left) — appears behind the Workspace tab; used by scaffold analysis (section 10).
  • Structure panel (right) — a large, exportable drawing of the selected molecule (PNG/SVG export buttons included).
  • Property Filters tab (left) — appears behind Workspace/Scaffolds; real-time min/max range sliders for every computed descriptor (section 5).
  • Conformers tab (right) — the interactive 3D viewer for generated conformer ensembles (section 11).
  • 3D Viewer tab (right) — an on-the-fly 3D view of whatever molecule is selected, no generation step needed (section 11).
  • Status bar (bottom) — messages, the progress bar, and the Cancel button during batch runs.
  • Look & Feel — 8 themes to choose from; press Ctrl+T any time to cycle to the next one (section 11).
  • View menu — show or hide any panel (Workspace, Structure, Scaffolds, Property Filters, Conformers, 3D Viewer, Chemical Space, Reproducibility). Closed a panel by accident? It's here.

4. Opening & converting molecule files

What you can open:

FormatExtensionsTypically contains
SDF.sdfMany molecules + data fields (the industry workhorse)
MOL.molA single molecule
SMILES.smi, .smiles, .txtOne molecule per line: SMILES name

How: File → Open Molecules…, or drag one or more files anywhere onto the window. Files are loaded one at a time with progress; loading more files adds to the table (it does not replace what's there).

If some records are broken (corrupt entries, unreadable SMILES), the good ones still load — a summary dialog lists exactly which entries failed and why. One bad molecule never aborts a load.

Converting formats: WaweKit features a built-in file format converter. Go to File → Convert Format… (Ctrl+Shift+V) to convert between CSV, SDF, MOL, and SMILES formats without loading them into the main workspace. Select the input file, choose the target output format, specify the output location, and click Convert. A conversion report lists total records processed, successes, and syntax warnings (such as truncating a multi-molecule dataset to a single-molecule MOL format).

5. Filtering the table (quick-filter & property filters)

The box above the table understands two kinds of input:

You typeYou get
amineRows whose name or SMILES contains "amine"
MW < 500Molecules lighter than 500 (needs descriptors computed)
LogP >= 2LogP of at least 2
Lipinski = 0No Rule-of-5 violations at all
Sim >= 0.7Similarity score ≥ 0.7 (after a similarity search)

Numeric tokens: MW, LogP, TPSA, HBD, HBA, RotB, Rings, Lipinski, Sim. Operators: < <= > >= = !=. Clear the box to remove the filter. Rows without the value yet are hidden by numeric filters — compute descriptors first.

Property Filters Panel

In addition to the quick-filter box, WaweKit provides an interactive Property Filters panel (tabbed with Workspace and Scaffolds on the left side of the window). Once you compute descriptors (Ctrl+D), this panel automatically discovers the minimum and maximum ranges of the active dataset and constructs range inputs for each property. Adjust the min/max values to filter the table rows in real-time. Clearing the dataset or recalculating descriptors automatically updates the filter ranges.

6. Standardize — clean up your structures

What it does: real-world files are messy — salts (·HCl), charged forms, duplicate entries, inconsistent tautomers. Standardization cleans every structure the same way so later numbers are comparable.

When to use it: right after loading, before computing anything else.

How:

  1. Chemistry → Standardize… (Ctrl+Shift+S).
  2. Tick the steps you want: strip salts (keep the largest fragment), neutralize charges, canonicalize tautomers, remove duplicates.
  3. Click OK. A report dialog then lists every change made to every molecule — nothing happens silently.

Good to know: standardizing replaces the dataset, so earlier computed values (descriptors, fingerprints, maps) are cleared — recompute them on the cleaned structures. That is deliberate: numbers computed on dirty structures should not survive next to clean ones.

PAINS & Drug-Likeness Alerts

WaweKit automatically audits loaded structures for common biological assay interference and toxicity alerts (using standard PAINS, Brenk, and NIH catalogs). Molecules triggering structural alerts display a warning badge (e.g. ⚠️ 1) in the Alerts column. Hovering over the cell displays a detailed list of the triggered warnings. Additionally, selecting a molecule with alerts displays details of the warnings in the Structure panel next to the molecular formula.

8. Fingerprints — preparing for similarity work

What it does: encodes each structure as a "bar-code" of bits so molecules can be compared quickly. Similarity search, chemical space, and clustering all build on fingerprints.

Which kind?

  • Morgan (default, radius 2, 2048 bits) — the field's standard choice. Use this unless you have a reason not to.
  • MACCS — 166 fixed yes/no structural questions; interpretable.
  • RDKit — path-based alternative.

How: Chemistry → Compute Fingerprints… (Ctrl+Shift+F), pick the kind, OK. A Fingerprint column appears (e.g. "Morgan · 24 on").

Good to know: you usually don't need this menu at all — similarity search, chemical space and clustering compute fingerprints automatically if missing. Two fingerprints built with different settings are never mixed: WaweKit recomputes rather than silently comparing incomparable codes.

9. Similarity search — "find more like this one"

What it does: ranks your whole dataset by how similar each molecule is to one query molecule — the classic "my compound works, what else do I have that looks like it?" question.

How:

  1. Either right-click a molecule → similarity search, or Chemistry → Similarity Search… (Ctrl+Shift+M) and paste any SMILES as the query (it doesn't have to be in your dataset).
  2. Pick the metric — Tanimoto is the standard (published thresholds like "≥ 0.85 = very similar" assume it).
  3. Click Search. The table sorts best-match-first; the query row is shown in bold if it's in the dataset.

Reading the numbers: 1.00 = identical fingerprint; above ~0.85 = very similar; below ~0.3 = mostly unrelated. Hover the Similarity column for full context ("Tanimoto vs aspirin · Morgan r2 · 2048b") — a score is only meaningful against that context. Keep just the good hits with a filter like Sim >= 0.7.

10. Scaffold analysis — group by core structure

What it does: strips each molecule down to its ring framework (its Bemis–Murcko scaffold) and groups the dataset by it — answering "how many distinct chemical series do I actually have?"

How:

  1. Chemistry → Analyze Scaffolds (Ctrl+Shift+A).
  2. The Scaffolds panel (left, behind Workspace) opens: a diversity headline plus every scaffold ranked by how many molecules share it.
  3. Click a scaffold to filter the table to just its members. Click it again (or Clear) to unfilter.
  4. Toggle Exact / Generic: generic ignores atom types, so benzene and pyridine cores group together — coarser families.

Good to know: molecules with no rings are grouped as "(acyclic)" — they have no scaffold, and that itself is worth knowing.

11. Conformers & Standalone 3D Viewer — 3D shapes

What it does: molecules are flexible — the flat drawing is only a summary. This generates several plausible 3D shapes (conformers) per molecule, ranks them by energy, and shows them in a rotatable 3D viewer.

How:

  1. Select the molecules you want in the table (3D generation is the heaviest computation — with nothing selected it runs on everything).
  2. Chemistry → Generate Conformers… (Ctrl+Shift+C). The defaults (10 conformers, MMFF94) are sensible; just click OK.
  3. The Conformers tab (right) opens: drag to rotate, scroll to zoom. The table below lists each conformer's energy, ΔE from the best, and RMSD; click one to view it.
  4. Export SDF saves all 3D coordinates for use in docking or other tools.

Good to know: the lowest-energy conformer is listed first. If MMFF94 can't handle a molecule, WaweKit automatically falls back to UFF and says so.

Standalone 3D Viewer

Beside conformer ensembles, you can view any selected molecule in interactive 3D dynamically using the 3D Viewer panel (tabbed with Structure and Conformers on the right). When a molecule is selected in the table, the 3D Viewer generates an on-the-fly 3D conformation (using RDKit's ETKDGv3 and MMFF optimization) so you can immediately view and rotate its structure. Use the dropdown inside the dock to switch representation styles (Ball & Stick, Stick, Space-fill, Wireframe).

Look & Feel Themes

You can customize WaweKit's visual style. Choose from 8 named themes matching your visual preferences: Night (default dark), Graphite, Gray, Moderate, Light, Creme Coffee, Sahara, and Nebula. Select these via Help → Look & Feel, or press Ctrl+T to cycle to the next theme. Links and text automatically adjust to remain fully legible across all themes.

12. Chemical space — a map of your dataset

What it does: places every molecule as a point on a 2D map where similar molecules land near each other — the fastest way to see your dataset's structure: its families, its outliers, its coverage.

How:

  1. Chemistry → Chemical Space… (Ctrl+Shift+P).
  2. Pick the method: PCA (fast, stable, distances meaningful globally) or t-SNE (slower, better at separating clusters; distances between far-apart groups are not meaningful).
  3. The map opens along the bottom. Hover a point to identify it; click or lasso-drag to select — the table selection follows, and vice versa.
  4. Colour by any descriptor (or cluster id) with the dropdown — e.g. colour by LogP to see where the greasy molecules live.
  5. The toolbar exports the plot as PNG/PDF/SVG for slides and papers.

13. Clustering — find the families

What it does: automatically groups similar molecules and writes each group's number into a Cluster column. Where scaffold analysis groups by exact core, clustering groups by overall similarity.

How:

  1. Chemistry → Cluster Molecules… (Ctrl+Shift+K).
  2. Pick the method:
    • Butina — the cheminformatics classic. You set a similarity cutoff (e.g. 0.35); the algorithm decides how many clusters exist. Lower cutoff → fewer, looser clusters.
    • K-Means — you say how many clusters you want; it finds the best split into exactly that many.
  3. Cluster 0 is always the largest family. Sort the Cluster column to read the dataset family-by-family.
  4. If a chemical-space map is open, it automatically recolours by cluster — each family lights up in its own colour (the picture above).

14. Substructure search — "which molecules contain this fragment?"

What it does: finds every molecule containing a fragment you specify, and highlights the matching atoms in every thumbnail and in the Structure panel.

How:

  1. Chemistry → Substructure Search… (Ctrl+Shift+U).
  2. Type the fragment as SMILES (a real molecule piece, e.g. c1ccncc1 for a pyridine ring) or SMARTS (a pattern language, e.g. [OH] for any hydroxyl). The dialog validates as you type and won't search an invalid query.
  3. Tick "show only matches" to filter the table down to hits, or leave it off to just mark them (✓ column).

Good to know: the ✓ column shows "✓ 2" when the fragment occurs twice in one molecule. The highlight stays until your next search or until the dataset changes.

15. Batch processing — run a whole pipeline

What it does: runs several steps back-to-back over the whole dataset — standardize → descriptors → fingerprints → scaffolds → clustering → export — so a routine workflow is one dialog, not six menus.

How:

  1. Chemistry → Batch Processing… (Ctrl+B).
  2. Tick the steps you want. Steps always run in the fixed, chemically sensible order shown.
  3. Optionally export the result to CSV (spreadsheet: names, SMILES, all computed values) or SDF (structures + values as data fields, for other chemistry software).
  4. Long batch? A Cancel button appears in the status bar. Cancelling stops cleanly at the next safe point — work finished so far is kept.

16. Reports — share your results

What it does: produces a polished, self-contained HTML and/or PDF summary of your dataset: headline stats, a descriptor min/mean/max table, Lipinski pass count, and a grid of structures. The HTML file is a single file — email it, drop it in a shared folder, or print it; no software needed to view it.

How:

  1. File → Generate Report… (Ctrl+R).
  2. Set a title, tick HTML and/or PDF, choose where to save.
  3. Compute descriptors before reporting if you want the statistics table included — the report shows what has been computed and stays silent about what hasn't.
  4. When it finishes, click Open to view immediately.

17. Settings

File → Settings… (Ctrl+,):

  • Theme — any of the 8 Look & Feel styles (section 11); also cyclable any time with Ctrl+T, and selectable directly from Help → Look & Feel. Your choice is remembered across restarts.
  • Log level — how much detail the log file records (Debug helps when reporting a problem).
  • Remember window layout — restore window size and panel arrangement on next start.

Settings persist in a small human-readable file in your user config folder; Restore Defaults puts everything back.

18. Research: Reproducibility Audit

What it does: a research-grade diagnostic unique to WaweKit. Different software cleans ("standardizes") the same molecule differently — so the same compound can end up with different identities in different databases, silently. The audit runs three standardization recipes over your dataset and reports:

  • Reproducibility scores — what fraction of your molecules come out identical under all recipes (measured two ways: by canonical SMILES and by InChIKey — they can genuinely disagree, which is itself a finding).
  • Agreement heatmap — which recipe pairs agree most.
  • Cause chart — which cleaning step (salt removal, charge neutralization, tautomer choice…) is responsible for each disagreement.
  • Labile-molecule list — exactly which of your molecules are pipeline-dependent. If one of them is your lead compound, you want to know.

How: Research → Reproducibility Audit… (Ctrl+Shift+R), keep all three protocols ticked, OK. "Attribute causes" is slower but produces the cause chart. Protocols that fail on a molecule are reported as failures, separately — a failure is not counted as divergence.

19. Plugins

WaweKit can be extended by third-party plugins. Installing one is just pip install <plugin> in the same Python environment — on next start it appears in the Plugins menu automatically. A misbehaving plugin is skipped with a log message; it can never prevent WaweKit from starting. Developers: see examples/example-plugin/ in the repository for a complete working template.

21. Tips & troubleshooting

  • A menu item is greyed out. Either no molecules are loaded yet, or another operation is still running — wait for the progress bar to finish.
  • My numeric filter shows nothing. Numeric filters need the value to exist: compute descriptors first (Ctrl+D). Rows without the value are hidden, not treated as zero.
  • Some molecules were "skipped" or "failed". That's by design — WaweKit processes everything it can and gives you the exact list of what it couldn't, with reasons, behind the Details button. One bad structure never stops a run.
  • Similarity scores look different than yesterday. Check the column tooltip: the score depends on the query and the fingerprint settings. Same settings → same numbers, always.
  • The 3D view is slow on huge datasets. Select a handful of molecules first — conformers run on the selection when there is one.
  • Where is the log file? The path is printed at startup and shown in the log; include it when reporting a problem. Set log level to Debug in Settings for maximum detail.
  • Something else? Open an issue on the project page — github.com/waweai/WaweKit (also linked from Help → About) — a screenshot plus the log file makes bugs fast to fix.
